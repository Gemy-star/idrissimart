# إصلاح Slugs العربية - دليل سريع 🚀

## تم إنشاء الملفات التالية:

### 1. أمر الإدارة (Management Command)
📁 الموقع: `main/management/commands/fix_arabic_slugs.py`

هذا هو الأمر الرئيسي الذي يقوم بإصلاح جميع slugs للفئات والإعلانات.

### 2. التوثيق الكامل
📁 الموقع: `docs/FIX_ARABIC_SLUGS_COMMAND.md`

يحتوي على شرح مفصل لكيفية عمل الأمر وجميع الخيارات المتاحة.

### 3. سكريبت تفاعلي
📁 الموقع: `fix_slugs.sh`

سكريبت bash تفاعلي لتسهيل استخدام الأمر.

---

## الاستخدام السريع ⚡

### الطريقة الأولى: استخدام السكريبت التفاعلي (موصى به للمبتدئين)
```bash
./fix_slugs.sh
```
سيظهر لك قائمة تفاعلية لاختيار ما تريد فعله.

### الطريقة الثانية: استخدام أوامر Django مباشرة

#### 1️⃣ معاينة التغييرات أولاً (Dry Run) - موصى به!
```bash
poetry run python manage.py fix_arabic_slugs --dry-run
```

#### 2️⃣ إصلاح الفئات فقط
```bash
poetry run python manage.py fix_arabic_slugs --categories-only --dry-run
```

#### 3️⃣ إصلاح الإعلانات فقط
```bash
poetry run python manage.py fix_arabic_slugs --ads-only --dry-run
```

#### 4️⃣ تطبيق التغييرات فعلياً (بدون dry-run)
```bash
poetry run python manage.py fix_arabic_slugs
```

## ما الذي يفعله الأمر؟ 🤔

1. **يقرأ جميع الفئات والإعلانات** من قاعدة البيانات
2. **يولد slugs جديدة** من الأسماء/العناوين العربية باستخدام `slugify(allow_unicode=True)`
3. **يتحقق من عدم التكرار** ويضيف أرقاماً عند الحاجة
4. **يحدث الـ slugs** في قاعدة البيانات
5. **يعرض تقريراً** بعدد التغييرات المنفذة

## نصائح هامة ⚠️

1. **استخدم Dry Run أولاً** - دائماً جرب بـ `--dry-run` قبل التطبيق الفعلي
2. **خذ نسخة احتياطية** - احفظ نسخة من قاعدة البيانات قبل التشغيل
3. **تحقق من النتائج** - راجع المخرجات للتأكد من صحة التغييرات
4. **اختبر الروابط** - تأكد من عمل الفلترة بعد التغيير

## استكشاف الأخطاء 🔧

### المشكلة: الأمر لا يعمل
**الحل**: تأكد من تفعيل البيئة الافتراضية:
```bash
poetry shell
python manage.py fix_arabic_slugs --help
```

### المشكلة: لا توجد تغييرات
**الحل**: ربما الـ slugs صحيحة بالفعل! راجع رسالة "already correct"

### المشكلة: أخطاء في قاعدة البيانات
**الحل**: الأمر يستخدم transactions لضمان سلامة البيانات، إذا حدث خطأ يتم التراجع تلقائياً

---

## للمطورين 👨‍💻

الكود يستخدم:
- `django.utils.text.slugify` مع `allow_unicode=True`
- `django.db.transaction.atomic()` لضمان سلامة البيانات
- معالجة دفعية (batch processing) للأداء الأفضل
- معالجة الأخطاء الشاملة

## الدعم 💬

3. تأكد من صحة إعدادات UTF-8 في قاعدة البيانات
