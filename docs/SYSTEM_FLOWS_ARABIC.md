## 1. تسجيل المستخدم والباقة المجانية

### الخطوات بالترتيب:

```
المستخدم يسجل حساب جديد
        ↓
النظام يعيّن نوع الحساب = "عادي" (DEFAULT)
        ↓
هل التحقق مطلوب للباقة المجانية؟
   ├── نعم → هل تحقق من الإيميل والموبايل؟
   │         ├── نعم → انتقل للخطوة التالية
   │         └── لا  → إرسال إشعار "حقق حسابك أولاً"، تأجيل الباقة
   └── لا  → انتقل مباشرة للخطوة التالية
        ↓
البحث عن باقة افتراضية (is_default=True) في النظام
   ├── موجودة → منح الباقة للمستخدم (ads_remaining = عدد إعلانات الباقة)
   └── غير موجودة → منح إعلان واحد مجاني لمدة سنة (احتياطي)
        ↓
إرسال إشعار ترحيب + بريد إلكتروني ترحيبي
```

### ما تحتويه الباقة المجانية:
| الحقل | الوصف |
|---|---|
| `ad_count` | عدد الإعلانات المسموح بها |
| `duration_days` | صلاحية الباقة (مثلاً 365 يوم) |
| `ad_duration_days` | مدة ظهور الإعلان الواحد (مثلاً 30 يوم) |
| `feature_*_price` | أسعار إضافات التمييز (في الباقة المجانية = صفر) |

### إعدادات التحكم (من لوحة المشرف):
- **SiteConfiguration** → `require_verification_for_free_package`: هل يجب التحقق قبل الحصول على الباقة المجانية؟
- **SiteConfiguration** → `require_email_verification`: إلزامية التحقق من الإيميل
- **SiteConfiguration** → `require_phone_verification`: إلزامية التحقق من الموبايل

## 2. إنشاء إعلان جديد

### الخطوات التفصيلية:

```
المستخدم يضغط "إنشاء إعلان"
        ↓
هل مسجل الدخول؟
   └── لا → تحويل لصفحة تسجيل الدخول
        ↓
النظام يفحص: هل لديه إعلانات مجانية متبقية؟
   ├── نعم (ads_remaining > 0) → رسوم النشر = صفر
   └── لا  → حساب رسوم النشر (انظر قسم التسعير)
        ↓
المستخدم يملأ نموذج الإعلان
(العنوان، الوصف، السعر، الصور، القسم، التفاصيل الإضافية)
        ↓
اختيار الإضافات المدفوعة (اختياري):
   • تمييز الإعلان (Highlighted)
   • إعلان عاجل (Urgent)
   • تثبيت الإعلان (Pinned)
   • إخفاء السعر "تواصل للعرض" (Contact for Price)
   • تحديث تلقائي يومي (Auto Refresh)
   • إضافة فيديو (Add Video)
        ↓
حفظ الإعلان مؤقتاً كـ DRAFT (مسودة)
        ↓
التحويل إلى صفحة الدفع
```

### ما يحدث في صفحة الدفع:

```
المبلغ الإجمالي = صفر؟
   ├── نعم (مجاني بالباقة) → زر "تفعيل مجاني" → خصم إعلان من الباقة
   └── لا (مدفوع) → اختيار طريقة الدفع:
            ├── دفع إلكتروني (PayMob / بطاقة)
            └── دفع يدوي (InstaPay / محفظة إلكترونية)
        ↓
بعد إتمام الدفع أو التفعيل المجاني:
        ↓
تحديد حالة الإعلان بناءً على نوع الحساب (انظر الجدول أدناه)
```

### حالة الإعلان بعد الدفع حسب نوع الحساب:

| نوع الحساب | حالة الإعلان | يحتاج موافقة؟ |
|---|---|---|
| **عادي** (DEFAULT) | قيد المراجعة (PENDING) | ✅ نعم |
| **ناشر** (PUBLISHER) | نشط (ACTIVE) | ❌ لا |
| **موثق** (VERIFIED) + القسم لا يشترط الموافقة | نشط (ACTIVE) | ❌ لا |
| **موثق** + القسم يشترط الموافقة | قيد المراجعة (PENDING) | ✅ نعم |
| **مشرف/طاقم** (Staff) | نشط (ACTIVE) | ❌ لا |

## 3. نظام الدفع والتسعير

### هرم التسعير - رسوم نشر الإعلان الأساسية:

```
أولاً: هل للمستخدم باقة بإعلانات متبقية؟
   └── نعم → رسوم النشر = 0 (مجاني من الباقة)

ثانياً (إذا لا باقة): هل للقسم المختار سعر خاص؟
   └── نعم → استخدم سعر القسم (ad_creation_price)

ثالثاً: هل للقسم الأب سعر؟
   └── نعم → وِرث السعر من القسم الأب

رابعاً (احتياطي): استخدم السعر الافتراضي للموقع (ad_base_fee)
```

### تسعير إضافات التمييز:

| الإضافة | المصدر عند وجود باقة | المصدر بدون باقة |
|---|---|---|
| تمييز (Highlighted) | سعر الباقة | `FEATURE_HIGHLIGHTED_PRICE` (افتراضي 50) |
| عاجل (Urgent) | سعر الباقة | `FEATURE_URGENT_PRICE` (افتراضي 30) |
| تثبيت (Pinned) | سعر الباقة | `FEATURE_PINNED_PRICE` (افتراضي 75) |
| تحديث تلقائي | سعر الباقة | `FEATURE_AUTO_REFRESH_PRICE` (افتراضي 35) |
| إضافة فيديو | سعر الباقة | `FEATURE_ADD_VIDEO_PRICE` (افتراضي 25) |
| إخفاء السعر | سعر الباقة | `FEATURE_CONTACT_FOR_PRICE` (افتراضي 0) |

> **ملاحظة:** في الباقة المجانية (is_default=True، price=0)، جميع أسعار الإضافات = 0

### أين تُضبط الأسعار؟

| الإعداد | الموقع |
|---|---|
| السعر الأساسي الافتراضي للإعلان | SiteConfiguration → `ad_base_fee` |
| سعر تمييز الإعلان الافتراضي | SiteConfiguration → `featured_ad_price` |
| سعر تثبيت الإعلان الافتراضي | SiteConfiguration → `pinned_ad_price` |
| أسعار الإضافات التفصيلية | django-constance → `FEATURE_*_PRICE` |
| سعر نشر إعلان في قسم معين | Category → `ad_creation_price` |
| أسعار الإضافات لباقة معينة | AdPackage → `feature_*_price` |

### طرق الدفع المتاحة:
- **دفع إلكتروني**: PayMob (بطاقة ائتمان/مدى) — يُفعَّل/يُعطَّل من `allow_online_payment`
- **دفع يدوي**: InstaPay / محفظة إلكترونية — يُفعَّل/يُعطَّل من `allow_offline_payment`

## 4. تعديل الإعلان وإعادة المراجعة

### متى يحتاج التعديل لموافقة المشرف؟

```
المستخدم يعدّل إعلانه
        ↓
هل المستخدم مشرف أو طاقم؟
   └── نعم → حفظ مباشرة بدون مراجعة
        ↓
هل الإعلان كان "نشط" (ACTIVE)؟
   └── نعم → فحص ما إذا تغيّر:
            • العنوان
            • الوصف
            • السعر
            • الصور
        ↓
هل تغيّر أي من هذه؟
   ├── نعم → تغيير حالة الإعلان إلى "قيد المراجعة" (PENDING)
   │         تعيين is_resubmitted = True
   │         إشعار المشرف (داخل التطبيق + بريد إلكتروني)
   │         رسالة للمستخدم: "تم إرسال التعديلات للمراجعة"
   └── لا  → حفظ مباشرة بدون تغيير الحالة
```

### ما لا يمكن تعديله:
- **القسم (Category)**: لا يمكن تغيير قسم الإعلان بعد النشر → يجب حذف الإعلان وإنشاء جديد

### إشعارات التعديل للمشرف:
- إشعار داخلي لجميع المشرفين
- بريد إلكتروني على عنوان `ADMIN_NOTIFICATION_EMAIL` (من django-constance)
- يتضمن: اسم المستخدم، عنوان الإعلان، رابط المراجعة، وقت التعديل

## 5. مراجعة وموافقة المشرف

### حالات الإعلان الممكنة:

```
DRAFT (مسودة)
   → مستخدم أنشأ إعلان لكن لم يتم الدفع بعد

PENDING (قيد المراجعة)
   → بعد الدفع: ينتظر موافقة المشرف
   → بعد التعديل: ينتظر إعادة المراجعة
   → بعد التجديد: ينتظر إعادة التفعيل

ACTIVE (نشط)
   → المشرف وافق، أو
   → تفعيل تلقائي (ناشر / موثق + قسم بدون موافقة)

REJECTED (مرفوض)
   → المشرف رفض مع ذكر السبب

EXPIRED (منتهي)
   → انتهت مدة الإعلان
```

### إجراءات المشرف:

| الإجراء | النتيجة | إشعار المستخدم |
|---|---|---|
| **الموافقة** (Approve) | PENDING → ACTIVE | إشعار داخلي + بريد إلكتروني + SMS |
| **الرفض** (Reject) | PENDING → REJECTED | إشعار داخلي + بريد إلكتروني + SMS مع سبب الرفض |

### إشعارات تلقائية للمشرف:
- عند إنشاء إعلان جديد: إذا كان `NOTIFY_ADMIN_NEW_ADS = True`
- عند تعديل إعلان: إشعار فوري بالتفاصيل
- عند الإعلانات المعلقة: إذا كان `NOTIFY_ADMIN_PENDING_REVIEW = True`

## 6. الحساب الموثق والنشر التلقائي

### مستويات التحقق:

```
UNVERIFIED (غير موثق)
   → حالة الافتراضية لكل مستخدم جديد
   → يمكنه نشر إعلانات لكن تحتاج موافقة مشرف

PENDING (قيد التحقق)
   → قدّم طلب توثيق مع وثيقة رسمية
   → ينتظر مراجعة المشرف

VERIFIED (موثق)
   → المشرف أقر التوثيق
   → يستفيد من النشر التلقائي (بشرط إعدادات القسم)
```

### مسار التوثيق:

```
المستخدم يرفع وثيقة التحقق
        ↓
حالة التحقق → PENDING (قيد الانتظار)
        ↓
هل رسوم التوثيق مفعّلة؟ (VERIFICATION_FEE_ENABLED)
   └── نعم → يجب الدفع أولاً
              ↓
   هل VERIFICATION_AUTO_APPROVE_ON_PAYMENT = True؟
   └── نعم → التوثيق تلقائياً بعد الدفع
        ↓
المشرف يراجع الوثيقة
        ↓
الموافقة → حالة التحقق = VERIFIED
        ↓
المستخدم يحصل على مزايا الحساب الموثق
```

### مزايا الحساب الموثق:

| الميزة | الشرط |
|---|---|
| نشر إعلانات بدون موافقة مشرف | الحساب موثق + القسم `require_admin_approval = False` |
| كتابة تقييمات | الحساب موثق |
| نشر وظائف | الحساب موثق + نوع الحساب شركة/تاجر |

### الناشر (PUBLISHER) — مستوى أعلى:

```
المستخدم عادي يشتري باقة ناشر أو يُرقَّى من قِبل المشرف
        ↓
نوع الحساب → PUBLISHER
        ↓
جميع الإعلانات الجديدة تُفعَّل تلقائياً (ACTIVE) بدون مراجعة
        ↓
حتى بدون توثيق الحساب
```

> **الفرق:** الموثق (VERIFIED) يحتاج إعدادات القسم، أما الناشر (PUBLISHER) فيُفعَّل دائماً.

## 7. تجديد الإعلان وانتهاء الصلاحية

### عند انتهاء صلاحية الإعلان:

```
الإعلان ينتهي (EXPIRED)
        ↓
المستخدم يختار تجديد الإعلان
        ↓
حساب رسوم التجديد
        ↓
الدفع (إلكتروني أو يدوي)
        ↓
بعد الدفع:
   الإعلان → PENDING (قيد المراجعة للتجديد)
   المشرف يوافق → ACTIVE من جديد
```

### تجديد الإعلانات المجانية من الباقة:
- إذا تبقت إعلانات في الباقة → يمكن التجديد مجاناً
- إذا انتهت الباقة → يجب الدفع أو شراء باقة جديدة

## 8. ملخص الحالات والصلاحيات

### رحلة الإعلان الكاملة من البداية للنهاية:

```
┌─────────────────────────────────────────────────────┐
│              إنشاء إعلان جديد                       │
└─────────────────────────────────────────────────────┘
                      ↓
              [DRAFT - مسودة]
                      ↓ (بعد الدفع أو التفعيل المجاني)
              [PENDING - قيد المراجعة]
              /                    \
   موافقة المشرف              رفض المشرف
       ↓                            ↓
  [ACTIVE - نشط]          [REJECTED - مرفوض]
       ↓                            ↓
  المستخدم يعدّل           المستخدم يصحح ويعيد
       ↓
  [PENDING - إعادة مراجعة] (is_resubmitted=True)
       ↓
  موافقة المشرف
       ↓
  [ACTIVE - نشط مرة أخرى]
       ↓ (انتهاء المدة)
  [EXPIRED - منتهي]
       ↓ (تجديد + دفع)
  [PENDING - تجديد]
       ↓
  [ACTIVE - مجدد]
```

### ملخص إعدادات التحكم الرئيسية:

| الإعداد | الموقع | الوظيفة |
|---|---|---|
| `require_verification_for_free_package` | SiteConfiguration | هل التحقق شرط للباقة المجانية؟ |
| `require_phone_verification` | SiteConfiguration | التحقق من الموبايل إلزامي؟ |
| `require_email_verification` | SiteConfiguration | التحقق من الإيميل إلزامي؟ |
| `ad_base_fee` | SiteConfiguration | رسوم نشر الإعلان الأساسية |
| `require_admin_approval` | Category (لكل قسم) | هل هذا القسم يشترط موافقة مشرف؟ |
| `NOTIFY_ADMIN_NEW_ADS` | django-constance | إشعار المشرف عند كل إعلان جديد |
| `ENABLE_EMAIL_NOTIFICATIONS` | django-constance | تفعيل/تعطيل البريد الإلكتروني |
| `ENABLE_SMS_NOTIFICATIONS` | django-constance | تفعيل/تعطيل الرسائل النصية |
| `VERIFICATION_FEE_ENABLED` | django-constance | هل التوثيق بمقابل مادي؟ |
| `VERIFICATION_AUTO_APPROVE_ON_PAYMENT` | django-constance | توثيق تلقائي بعد الدفع؟ |
| `FEATURE_HIGHLIGHTED_PRICE` | django-constance | سعر تمييز الإعلان |
| `FEATURE_URGENT_PRICE` | django-constance | سعر الإعلان العاجل |
| `FEATURE_PINNED_PRICE` | django-constance | سعر تثبيت الإعلان |

## ملاحظات تقنية مهمة

1. **الباقة الافتراضية**: يجب أن توجد دائماً باقة واحدة بـ `is_default=True` في قاعدة البيانات، وإلا يحصل المستخدم على إعلان واحد فقط كاحتياطي.

2. **أسبقية الحالة**: عند إنشاء إعلان:
   - المسودة (DRAFT) لا تُغيَّر تلقائياً
   - الناشر (PUBLISHER) → نشط دائماً
   - الموثق (VERIFIED) + قسم بدون موافقة → نشط
   - العادي (DEFAULT) → قيد المراجعة دائماً
   - الطاقم/المشرف → نشط دائماً (ما لم يكن مسودة)

3. **التسلسل الهرمي للتسعير**: القسم الفرعي يرث سعر القسم الأب إذا لم يكن له سعر خاص، وإذا لم يكن للقسم الأب سعر يُستخدم السعر الافتراضي للموقع.

4. **الإعلانات المعاد تقديمها**: `is_resubmitted=True` يُمكَّن المشرفَ من التعرف على الإعلانات المُعدَّلة التي تحتاج إعادة مراجعة.

5. **خصم الإعلانات من الباقة**: يتم الخصم فقط عندما يكون `base_fee = 0` (أي استخدام الباقة، وليس الدفع الفردي).
